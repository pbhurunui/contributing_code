2. INIT: Project Goal

Welcome to your final project! The goal is to use all the skills you've learned in Python, HTML, CSS, and JavaScript to create a working prototype of a simple digital app or mini-game. This is your chance to be creative and build something from scratch. This section outlines the main choices you have for your project.

Option 1: Mini-Game Prototype üëæ

Build a simple, fun, web-based game. Think about what makes a game enjoyable and try to create a single, complete interaction.

Examples:

  • A number guessing game
  • A simple memory card flipper
  • A single-screen quiz with instant feedback

Option 2: Interactive Application üíæ

Create a basic utility application that helps a user accomplish a task. Focus on making it useful and easy to understand.

Examples:

  • An enhanced calculator with extra functions
  • A simple data visualizer
  • An expanded to-do list with a delete function

4. SUBMIT: The Deliverables

Your final mark is based on three key components. This section breaks down exactly what you need to plan, build, and present. Pay close attention to the weighting for each part, as planning and presenting are just as important as the code itself.

20%

A. Planning & Design

  • Wireframe of your layout
  • Algorithm or Flowchart of your logic
  • Feature list (mandatory & extensions)
70%

B. Final Working Code

  • The complete, functional prototype
  • Clean, structured, and commented code
10%

C. Presentation & Reflection

  • A 3-5 minute demonstration
  • Explanation of key code concepts
  • Reflection on your process

Code Structure & Readability (30%) Code is copied with minimal understanding; structure is random or nonexistent. Code shows sequential attempts at using different structures; commenting is inconsistent. Code is logically organised, consistently formatted, and uses meaningful variable/function names. Design & User Experience (UX) (20%) The interface is difficult to use, and only basic styling is present. The design uses multiple style rules, but the aesthetic is inconsistent and not responsive.
